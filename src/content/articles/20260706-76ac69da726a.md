<h2 class="post__body__extend__item__title" data-v-7653d8d5="">Apple 创作坊，与 Pixelmator Pro 和 Apple Intelligence 深度集成</h2>
<ul><li>平台：macOS / iPadOS / iOS</li><li>关键词：Apple Creator Studio、Pixelmator、Final Cut Pro</li></ul>
<p>@ElijahLee：今年 1 月，Apple 推出了面向内容创作者的一站式订阅服务 Apple Creator Studio，将视频剪辑、音乐制作、图像编辑和办公创作工具整合到一个平台中。时隔半年，Apple 发布了首个重大更新。新版主要实现 Pixelmator Pro 深度融入文稿创作和视频剪辑的工作流程、Final Cut Pro 引入更智能的工具、新增形状和图像生成功能等等。</p>
<p>首先是视频编辑软件 Final Cut Pro 获得了多项的新功能。自动生成字幕（Generate Captions）可以利用设备端 AI，自动识别视频中的语音内容，并直接生成可编辑的字幕。生成后的字幕支持调整字体、颜色、位置等样式，大幅减少手动听打字幕的时间，非常适合 Vlog、访谈、课程视频等内容创作。</p>
<img src="https://cdnfile.sspai.com/2026/07/06/article/61b695530d1904b7a34fb78d167ee09d.jpeg?imageView2/2/w/1120/q/90/interlace/1/ignore-error/1" alt="" loading="lazy">
<p>在 macOS 端，智能蒙版（Auto Mask）功能利用设备端 AI，可以实现无需繁琐抠图，即可快速完成背景替换、局部调色等操作。目前支持自动识别人物、头发、天空、树叶等复杂元素。结合已有的磁性蒙版（Magnetic Mask）搭配使用，可以进一步扩充蒙版支持识别和调整的对象。</p>
<p>Pixelmator Pro 真正融入 Apple 创作坊多个 App 中，也是本次更新的一大亮点。现在，我们可以直接从 Final Cut Pro、Keynote、Pages、Numbers 这几个应用直接调用 Pixelmator Pro，实现在 Final Cut Pro 中编辑选定帧画面、在 Keynote 中修改演示图片、在 Pages 中处理插图等。编辑完成后即可返回原应用，无需手动导出、重新导入素材，使整个流程更加顺畅。</p>
<img src="https://cdnfile.sspai.com/2026/07/06/article/a709e0a758faecb206a84d9a170c76c9.png?imageView2/2/w/1120/q/90/interlace/1/ignore-error/1" alt="" loading="lazy">
<p>生成形状（Shape Generation）和生成图像（Image Generation）功能基于设备端 AI，可以使用户通过自然语言描述图像需求，即可生成各种矢量形状换和图像，对于经常制作演示文稿的人来说，可以省去寻找图片素材或自行绘制图形的时间。我们还可以选择图像风格、视图形状、宽高比等要求。例如，我在 Keynote 中描述图像内容为「办公室员工在小会议室讨论某公司的经营情况表格」，风格要求为写实，经过不到 1 分钟的时间，应用就生成了一张图像，效果出乎我的意料。生成功能可以在 Pixelmator Pro、Keynote、Pages 和 Numbers 中使用。此外，Pixelmator Pro 新增内容中心，其中包含各种 Apple 精选的摄影图片、插图和图形。</p>
<img src="https://cdnfile.sspai.com/2026/07/06/article/c61db0d7827e06ededf69b607f75308e.jpeg?imageView2/2/w/1120/q/90/interlace/1/ignore-error/1" alt="" loading="lazy">
<p>音乐创作软件 Logic Pro 也获得多项更新。和弦 ID 经重新构建，为和声分析带来更高的准确度。此外新增 Producer Project，将专业制作人的完整工程作为学习范例，对于学习音乐制作来说，相当于获得了一套可拆解研究的专业教材。</p>
<p>此次 Apple 创作坊更新以免费升级的形式提供给所有订阅用户，你可以在 App Store 免费下载。新订阅用户价格为每月 38 元、每年 380 元， 新订阅支持 1 个月免费试用。此外，购买新 Mac 或部分 iPad 的用户，还可以领取三个月免费试用。</p>

<h2 class="post__body__extend__item__title" data-v-7653d8d5="">Glaze：手搓一个 Mac 应用，从未如此简单</h2>
<ul><li>平台：macOS</li><li>关键词：AI 编程</li></ul>
<p>@Vanilla：如果你已经掌握了如何使用各类 Coding Agent，不管是国外的 Claude Code、Codex，还是国内的 DeepSeek、Kimi Code 等，都可以让毫无编程基础的人做出一款简单的 Mac 应用。虽然期间你会遇到各种各样的问题，消耗大量的 token，但是最终肯定能跑起来。那么，有没有更加简单的办法呢？有的朋友，那就是 Raycast 团队推出的一款新应用 —— Glaze，专注于通过 AI 来生成 Mac 应用。听起来不错，那真实的效果怎么样？接下来让我们一起来看看。</p>
<p>下载安装好 Glaze 后，我们就可以直接开始了，只需要在侧边栏的 My Projects 界面下点击右上角的「New Project」即可开始创建自己想要的应用了。在弹出的对话窗口中，直接输入对应用的功能描述，还可以上传素材文件，或者启用 Plan 模式，也可以关联 Github、Linear、Slack 等服务，用起来就和普通的 Coding 工具无异。</p>
<img src="https://cdnfile.sspai.com/2026/07/06/article/ea6de19ac62d7cfe8710469d5ac0f5dd.png?imageView2/2/w/1120/q/90/interlace/1/ignore-error/1" alt="" loading="lazy">
<p>接下去就是耐心的等待，在生成的过程中 Glaze 还会根据我的需求提出一些优化建议，并让我进行选择，而对话框最上方会显示 Plan 的进度，一步一步完成后，新的应用就会自动安装到我的 Mac 电脑上，同时它也会出现在我的 My Projects 界面中。</p>
<img src="https://cdnfile.sspai.com/2026/07/06/0273797964d6aa55a31c31f5d803f062.png?imageView2/2/w/1120/q/90/interlace/1/ignore-error/1" alt="" loading="lazy">
<p>以我自己创建的这个应用为例，它是一个功能非常纯粹的拼图工具，导入图片后就可以通过各项自定义选项来进行拼接，完成后导出到本地或者剪贴板。在 iOS 系统上，我一直都用 Picsew 来完成这项工作，但是在 Mac 上我一直没找到一款同样好用的免费工具，所以我打算自己做一个。没想到通过 Glaze，这么简单地就把它做出来，而且效果比我想象中还要好。</p>
<p>在第一版中，我发现一些按钮实现的功能报错了，而且导出的图片是圆角矩形，所以我需要让 Glaze 帮我修正。点击应用右下角的 Glaze agent 按钮，找到 Edit in Glaze 或者 Annotate 选项，我们就可以继续通过 Glaze 来 debug 了。</p>
<img src="https://cdnfile.sspai.com/2026/07/06/article/06be96ded74eb616ce73ea5000ad9a41.png?imageView2/2/w/1120/q/90/interlace/1/ignore-error/1" alt="" loading="lazy">
<p>最后一步，把这款我自己打造的应用发布出去。Glaze 支持通过 AI 来自动生成 tagline、full description、icon 等内容，然后一键发布到 Glaze 自己的应用商店。通过审核后，所有人都可以通过商店来下载安装我的应用了。</p>
<img src="https://cdnfile.sspai.com/2026/07/06/article/5030ac48b331167a42efc45fc186f891.png?imageView2/2/w/1120/q/90/interlace/1/ignore-error/1" alt="" loading="lazy">
<p>如果你不想公开发布自己的应用，那么也可以通过链接的方式来内部分享应用，不过这就需要订阅 Pro 或者 Team 付费计划，价格分别是每月 20 美元和 30 美元。Glaze 可以在官网免费下载体验，你会获得 120 个 credits，足够做出一个简单的应用了。像我做的这个拼图工具，就用掉了 80 个 credits。</p>
<p>总的来说，Glaze 在用户操作体验和最终交付效果上都做得相当不错，还打通了应用发布和共享的链路，对有团队应用开发需求的群体来说是一个不错的选择。</p>
<p>One more thing：本文所有的配图都使用我通过 Glaze 打造的 Image Joiner 拼接生成，欢迎大家试用。</p>
<h2 class="post__body__extend__item__title" data-v-7653d8d5="">Otty：老牌写作工具开发商推出终端工具</h2>
<ul><li>平台：macOS</li><li>关键词：终端、AI</li></ul>
<p>@化学心情下2：AI Agent 的普及让终端工具变得更为关键。Claude Code、OpenCode、Codex CLI 等工具均以命令行形态运行，在任务执行效率上表现突出，也使开发与信息处理进一步向终端集中。</p>
<p>但现有终端工具在多 agent 并行场景下仍有局限。以 macOS 为例，Ghostty 是较常见的新选择，但单窗口、单会话结构在同时运行多个 agent 时不够灵活。Otty 是由 Markdown 工具 Typora 开发团队推出的终端应用，针对这一使用场景做了重新设计。</p>
<img src="https://cdnfile.sspai.com/2026/07/06/article/7a192727bf20150ff6967b92a056c92a.png?imageView2/2/w/1120/q/90/interlace/1/ignore-error/1" alt="" loading="lazy">
<p>打开 Otty，首先看到的是较为克制的界面设计，默认主题是与 Typora 类似带有纸类质感的配色风格，左侧的侧边栏用于统一管理多个终端会话，可以同时运行不同任务或 agent。设置层面支持较完整的本地化配置，包括主题、布局与标签行为等。相比一些偏开发者取向的终端工具，Otty 的常用选项会更容易调整。</p>
<img src="https://cdnfile.sspai.com/2026/07/06/article/69f3f131584e35d66a3b10079a599719.png?imageView2/2/w/1120/q/90/interlace/1/ignore-error/1" alt="" loading="lazy">
<p>性能上，Otty 使用 Rust 构建，并在 macOS 上通过 Metal 进行 GPU 加速。在大段日志输出或频繁滚动场景下，界面响应也能够保持稳定。</p>
<p>标签系统是核心设计之一，除垂直标签栏外，还支持顶部与底部布局切换，并可调整新标签打开位置与侧边栏显示方式。标签支持分组，在同时运行多个 AI agent 或多个项目时便于管理。</p>
<img src="https://cdnfile.sspai.com/2026/07/06/article/f057ed4791862ecead0a17477f4c13f9.png?imageView2/2/w/1120/q/90/interlace/1/ignore-error/1" alt="" loading="lazy">
<p>在 AI agent 集成上，Otty 支持 Claude Code、Codex、OpenCode 等工具的运行状态识别。在标签页中启动后，可直接看到执行状态，如运行中、等待输入或已完成。每个会话侧边区域集成了 Git 状态、文件树与命令历史。Info 面板中可复制 session ID，或开启分支对话。</p>
<img src="https://cdnfile.sspai.com/2026/07/06/ed4d7481747315d26dd667026a3670d7.gif" alt="" loading="lazy">
<p>目前 Otty 仅提供了 macOS 版本，根据官网的信息显示，Windows 与 Linux 的版本还在开发当中。目前 Otty 可以免费使用，开发者也在官网表示之后可能围绕 AI 或云服务收费。</p>

<h2 class="post__body__extend__item__title" data-v-7653d8d5="">BetterCmdTab：可自定义化窗口管理</h2>
<ul><li>平台：macOS</li><li>关键词：窗口管理</li></ul>
<p>@Snow：macOS 平台的窗口管理类应用堪称「提效」典范，应用切换预览可以帮你快速锁定目标，多窗口分屏有助于你多线操作，有些应用还支持接管应用内标签页实现更深层级的窗口管理。去年底我曾推荐过一款集大成的 Wins，今天这款 BetterCmdTab 大概称得上它的绝佳平替，不仅核心功能基本可以轻松拿下，而且自定义程度更高，最重要的是，它完全免费。</p>
<p>BetterCmdTab 为窗口切换提供了 3 种模式，与原生应用切换器类似的大图标网格模式、与 TabTab 类似的垂直列表模式以及 Wins 同款的实时窗口预览。你可以在 Settings-Appearance 一栏中按需选择，前两种模式还支持三档图标和文字的显示大小调节。</p>
<img src="https://cdnfile.sspai.com/2026/07/06/article/75566c8e376d4e6bdd808f067f982d8d.png?imageView2/2/w/1120/q/90/interlace/1/ignore-error/1" alt="" loading="lazy">
<img src="https://cdnfile.sspai.com/2026/07/06/article/224045946aedcf13ad51b00ccc7b88a2.png?imageView2/2/w/1120/q/90/interlace/1/ignore-error/1" alt="" loading="lazy">
<p>在按下组合键 Command + Tab 激活切换器窗口后，你可以看到除了显示常规的图标、应用名或预览外，应用顶部还会显示一些字母组合，此时若你在键盘上输入对应组合前缀，无需切换选框或移动光标即可实现快速跳转。</p>
<img src="https://cdnfile.sspai.com/2026/07/06/24f6ed56def9df0c0f76caa3d60970cc.gif" alt="" loading="lazy">
<p>如果你开启的窗口较多，你还可以在这一页面按下 / 键，输入应用名中部分字符实现模糊查找跳转，如果想要调取的应用尚未启动，甚至可以借助它实现应用启动器类似的快速启动功能。</p>
<img src="https://cdnfile.sspai.com/2026/07/06/article/007b23c4e3ece91b30155eb979b1aad6.gif" alt="" loading="lazy">
<p>BetterCmdTab 同样支持读取部分应用内的标签页数据，如 Chrome、Safari、Arc、Finder、Terminal 等。在 Settings-Behavior-Tabs 中打开对应开关，应用就会自动将标签页拆分展现，无需跳转到对应应用，再去标签页中寻找对应页面。如果你的应用还未获得 BetterCmdTab 的标签页支持，还可以在切换器页面尝试按下 <code>\</code> 键，借助系统辅助功能和 Apple Events，部分三方应用也支持在切换器底部显示标签页预览条，你可以在这里实现快速选择并切换。如果以上均未获得支持，BetterCmdTab 还提供了 <code>Cmd + `</code> 这一组合键来实现应用内多窗口切换，你可以按需自行拆分窗口，然后利用这一组合快速切换。</p>
<img src="https://cdnfile.sspai.com/2026/07/06/741b213cbe0a786e00998740413e43d4.gif" alt="" loading="lazy">
<p>说到快速切换，就不得不提 BetterCmdTab 非常实用的「三指轻扫」手势，你同样可以在 Settings-Experimental 一栏中自定义，我习惯将其设置为 Quick Switch，启用后你就可以使用这一手势在最近激活的两个窗口实现快速切换，在 MacBook Air 13 这样屏幕空间有限的设备上，效率提升有时会优于分屏显示。</p>
<p>BetterCmdTab 同样支持利用快捷键实现多窗口分屏布局，你可以在 Settings-Windows 中自定义不同分屏模式以及隐藏或显示全部窗口的快捷键组合。该项设置中还有一个 Cycle tile width 的开关，启用后在左 / 右侧分屏时再次按下对应快捷键组合，可以让窗口保持贴合该侧的情况下，在 <strong>1/2 → 2/3 → 1/3 全屏</strong>宽度中依次循环切换。此外，如果你有一些重要窗口需要保持常显，则可以添加在显示列表中，避免批量操作后又得手动打开一部分窗口的麻烦事儿。</p>
<img src="https://cdnfile.sspai.com/2026/07/06/b7e8cca3f02af5722da3695301ed45e8.gif" alt="" loading="lazy">
<p>除了核心功能足够优秀，BetterCmdTab 还有高度个性化这一优势。你几乎可以在应用设置每一栏中都找到丰富的自定义项，例如在 Profiles 中为不同显示器设置独立的展现逻辑，在 Behavior 中调整切换器中的排列优先级，在 Apps 中标记置顶应用以及为不同应用配置独立的显示规则，在 Appearance 一栏中自定义展现方式、标题位置、浮窗透明度、圆角曲度、重点色等，你甚至还可以自定义回调近期关闭的多少个应用窗口。虽然应用不支持云端配置同步，但你可以在 General 一栏中导出和导入配置，如果有多设备需求，也不会被重复的配置操作劝退。</p>
<p>你可以在应用官网上免费下载 BetterCmdTab，或通过 Homebrew 快速安装。</p>
<h2 class="post__body__extend__item__title" data-v-7653d8d5="">Trayzero：采用 GTD 理念的任务管理工具</h2>
<ul><li>平台：Android</li><li>关键词：任务管理、GTD</li></ul>
<p>@Peggy_：曾几何时，GTD 就是任务管理的化身，不管你是否深入了解过何为 GTD，都不影响你当下使用的几乎所有任务管理工具都有它的影子。不过随着任务管理工具的发展和人们工具使用方式的改变，纯粹地遵循 GTD 理念的应用反倒日渐稀少了。今天要介绍的 Trayzero 则声称其复刻了 GTD 的完整流程和理念。</p>
<img src="https://cdnfile.sspai.com/2026/07/06/article/f13534e9785759a79e439b38f0e5db5e.jpeg?imageView2/2/w/1120/q/90/interlace/1/ignore-error/1" alt="" loading="lazy">
<p>进入应用后，默认界面便是收件箱，所有初步添加的任务都会被整理在这里以备下一步处理。随着任务数量的增加，我们需要抽出专门的时间来决定所有任务的下一步动作，Trayzero 会在应用首页给出醒目的提示，来引导我们处理收件箱内的所有任务。</p>
<img src="https://cdnfile.sspai.com/2026/07/06/article/10dad880e9d5f92ece4f90156a799bb9.jpeg?imageView2/2/w/1120/q/90/interlace/1/ignore-error/1" alt="" loading="lazy">
<p>跟随指引进入处理流程后，我们要做的便是简单的选择题，当下的任务是立刻就做还是留待以后完成，抑或是直接删除，对于需要完成的任务，我们还可以进一步判断是立刻去做还是稍后厘清后再完成，在这里所谓的「2 分钟法则」指的就是能立刻完成的任务，此类任务往往不必要长久停留在任务管理工具当中耗费心力，加上一步判定或许能有效促使我们即刻行动起来。</p>
<img src="https://cdnfile.sspai.com/2026/07/06/article/6ff9ab0f1d9f44f25bb5199f10a9100f.jpeg?imageView2/2/w/1120/q/90/interlace/1/ignore-error/1" alt="" loading="lazy">
<p>下一步判定则是选择任务是否需要由本人完成，这一步也符合我们日常工作当中遇到的实际情况，某些任务的完成取决于和他人之间的配合、或是干脆就需要督促他人来完成，而某些任务则需要亲力亲为，两类任务对于我们注意力和时间的损耗各不相同。最后，我们还要判断任务是否有必要拆分成更详细的步骤以及确切的完成时间。</p>
<img src="https://cdnfile.sspai.com/2026/07/06/article/55ce0386a2b74fa9e39d09d1609895cf.jpeg?imageView2/2/w/1120/q/90/interlace/1/ignore-error/1" alt="" loading="lazy">
<p>以上步骤看似复杂，但对于任务量多且零碎的人士而言，提前厘清永远好过做时再额外思考。当然具体使用上还是要看个人习惯，如果在任务管理工具本身上消耗的时间已经超过了完成任务所需的时间，也会显得本末倒置。毕竟，类似的工具随处可见，但是任务的处理最终还是由我们这样具体的人去完成。</p>
<img src="https://cdnfile.sspai.com/2026/07/06/article/38080606ff809ecf40b737101e997a1a.jpeg?imageView2/2/w/1120/q/90/interlace/1/ignore-error/1" alt="" loading="lazy">

<h2 class="post__body__extend__item__title" data-v-7653d8d5="">ObtainX：更现代化的 Obtainium 增强版</h2>
<ul><li>平台：Android</li><li>关键词：应用管理</li></ul>
<p>@大大大K：少数派以前介绍过 Obtainium 这款开源应用的更新追踪工具，它可以从 GitHub 等源头抓取更新，让 Android 用户们安装和管理开源 App 方便许多。不过原版工具还是有一些不太方便的操作，比如抓取应用繁琐、详情查看不清晰等。于是就有开发者基于原版产品进行了改进，推出了增强版的「ObtainX」。</p>
<p>作为原版的「超级加倍」，我认为 ObtainX 最大的改进就是提升了 App 的添加效率。原版 Obtainium 里添加追踪需要手动输入链接或者使用那个并不是很顺畅的「搜索」功能。但 ObtainX 直接支持了「从设备批量导入」功能，它能直接扫描系统里已经装好的应用（就像原来的 GitHub Store 那样），并在指定的仓库里搜索源地址开启更新追踪。在查看 App 详情时，ObtainX 支持查看更新版本、资源数量以及其它原地址的检索，比 Obtainium 要详细得多。</p>
<img src="https://cdnfile.sspai.com/2026/07/06/article/0a0befd6374563fa65894c37d2b6222e.png?imageView2/2/w/1120/q/90/interlace/1/ignore-error/1" alt="" loading="lazy">
<p>除了 App 导入更加方便外，ObtainX 在安全性上也有增强，毕竟这些开源 App 大部分都是我们自己寻找安装源，如果因为链路问题导致包被污染或替换就麻烦了。而 ObtainX 引入了安装包的构建校验，它会在下载时去跟 F-Droid 构建源码或者 GitHub Attestation Proof 做比对。</p>
<p>虽然它无法分析包内是否包含恶意程序，但至少可以确保下载的安装包和发布的源码相同，这已经在很大程度上保证了安全性。另一方面，ObtainX 支持与 App Manager 等第三方工具集成，安装包下载后可以调用这些工具进行更详细的权限和资源查看。</p>
<img src="https://cdnfile.sspai.com/2026/07/06/article/86fbcb6d2635cab7319554756a02efcd.png?imageView2/2/w/1120/q/90/interlace/1/ignore-error/1" alt="" loading="lazy">
<p>此外，ObtainX 把进阶功能也做得非常易用。比如原版里想要实现更精细化的版本管理，或者过滤掉某些特定版本，需要人工输入复杂的正则表达式。ObtainX 专门做了一个「正则表达式助手」按钮，可以基于当前已发行的版本号创建默认正则。另外，它还支持文件夹的自动归类，通过对应用名称、作者、包名等条件的筛选自动整理。</p>
<img src="https://cdnfile.sspai.com/2026/07/06/article/1a5a041f44033efecb0b251bb3c8e50a.png?imageView2/2/w/1120/q/90/interlace/1/ignore-error/1" alt="" loading="lazy">
<p>而在 UI 设计上，ObtainX 比原版也要好看太多，它从动态取色、控件等方面完整适配了 Material 3 设计，还可以开启极具质感的磨砂「渐变透明」效果，更让我惊喜的是，ObtainX 还支持「实时通知」功能，App 下载和等待安装过程中都有展示，毫不掩饰地说，这个功能足以让我卸载原版了。</p>
<img src="https://cdnfile.sspai.com/2026/07/06/article/46d46fae3b452299e8d9bd073455e5d7.jpeg?imageView2/2/w/1120/q/90/interlace/1/ignore-error/1" alt="" loading="lazy">
<p>ObtainX 绝不是一个简单的「换壳版」，它在优化了老版诸多体验硬伤的同时，还极大提升了易用性和使用体验，对于大屏设备和折叠屏设备也有适配。所以如果你也在使用 Obtainium 作为开源 App 管理工具，那我非常建议你尝试一下这款增强版。现在，你可以在 GitHub 和 F-Droid 免费下载 ObtainX。</p>
