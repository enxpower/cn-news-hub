<p>和 macOS 一样，Windows 也提供了一个应用商店，但和 macOS 不同的是，很多 Windows 软件不会选择通过应用商店来分发。所以过去我们需要使用各种各样的包管理器，比如 Scoop、Chocolatey，或者是直接找到软件官网下载安装。</p>
<p>这样的需求也在中国市场上催生了各种各样的软件管家，比如 360 软件管家、火绒软件管家、腾讯应用宝，因为这些软件背后的商业利益或者本身只管理通过其自己安装的软件，用起来也总觉得不是很完美。</p>
<p>直到我发现了一个接近完美的 Windows 应用商店：UniGetUI。</p>
<h2>UniGet UI 是什么？</h2>
<p>严格来说，UniGetUI 并不是一个应用商店。它是一个为 Windows 10、Windows 11 常用包管理器提供 GUI 的工具。得益于支持的包管理器足够多，你几乎可以将它当作「应用商店」或者软件管理工具来使用。</p>
<p>作为一个包管理器的 GUI 实现，它支持 Windows 近年来正火的 winget，老牌包管理器 Scoop 和 Chocolaty，同时也针对 Windows 下的 Powershell 5 和 Powershell 7 提供了相关的包管理器；编程用户常用的 npm、pip、cargo 和 vcpkg 也有涉猎。虽然能力不完全对齐，但最基本的安装、管理和卸载都是支持的。</p>

![](https://cdnfile.sspai.com/2026/06/17/article/f4d7de3c98996c5ee1b01d14a1ea801f.png?imageView2/2/w/1120/q/90/interlace/1/ignore-error/1)

<p>这些包管理器的上游来源组合在一起，基本上可以覆盖我们所有的日常软件使用了。你电脑上 90% 的软件在这里应该都可以找得到。</p>
<h2>如何使用 UniGetUI？</h2>
<h3>安装 UniGetUI</h3>
<p>UniGetUI 安装的方式有多种，你可以选择直接在 Microsoft Store 中安装。但如果你的 Microsoft Store 和我的一样时常抽风，也可以考虑使用 winget、Scoop、Chocolatey 来安装，或者直接下载安装包，就像装 macOS 一定先装 Homebrew 一样。
 </p>
<h3>安装软件</h3>
<p>安装完成后，接下来就比较简单了。直接打开软件，进入发现软件包，搜索你要使用的软件即可：</p>

![](https://cdnfile.sspai.com/2026/06/17/article/96a0599f07935bd361752e350f5ae48f.png?imageView2/2/w/1120/q/90/interlace/1/ignore-error/1)

![](https://cdnfile.sspai.com/2026/06/17/article/7a821caa19b9938fac1acfcaf883d25c.png?imageView2/2/w/1120/q/90/interlace/1/ignore-error/1)

<p>搜索完软件后，可以点击并在弹出的窗口中查看对应软件的描述信息，了解这个软件是否是你需要的，然后进行下一步。</p>

![](https://cdnfile.sspai.com/2026/06/17/article/02cb1c1b09b2175027f5cea169f5201d.png?imageView2/2/w/1120/q/90/interlace/1/ignore-error/1)

<p>不过你可能会发现「为什么我搜索到的软件不如你的多」？UniGet UI 毕竟是一个包管理器的 UI，他所能安装的软件的选项取决于你的电脑上有哪些包管理器。默认情况下，你的系统里会自带 winget，所以你搜到的都是 winget 来源中的软件包。如果你需要更多的软件，则需要你安装更多的包管理器并开启相关支持。</p>

![](https://cdnfile.sspai.com/2026/06/17/article/00ccc7e4d7ea531d45c612a23b4a51f4.png?imageView2/2/w/1120/q/90/interlace/1/ignore-error/1)

<h3>更新软件</h3>
<p>安装了一些软件后，接下来的日常就是更新软件和管理软件。</p>
<p>在 UniGetUI 当中这件事也变得非常简单，只需要点击左侧的菜单，软件会自动查询对应的软件版本和你已经安装的软件，你可以根据自己的需求选择你需要更新的软件。当前进度会展示在底部的队列中，你也可以修改配置来提升任务并发处理速度。</p>

![](https://cdnfile.sspai.com/2026/06/17/article/14524e9e61411c8aa9e5c82d3c2bc195.png?imageView2/2/w/1120/q/90/interlace/1/ignore-error/1)

![](https://cdnfile.sspai.com/2026/06/17/article/7408c5ee5dee1a490f6179c5a0d33ec3.png?imageView2/2/w/1120/q/90/interlace/1/ignore-error/1)

<h3>封装软件捆绑包</h3>
<p>除了上面的安装、更新、卸载软件之外，UniGetUI 还有一个不错的功能：制作软件捆绑包。</p>
<p>顾名思义，这是一个帮助你将一批软件打包成一个 Bundle 的能力。有了这个能力，我们就可以非常方便地创建出你自己的软件组合。对有一定软件使用习惯的人来说，你可以借助这个功能维护一套简单的软件 Bundle 并不断更新。这个 Bundle 将会在你下次配置新电脑或重装系统之后，作为一个方便的快速启动包来完成初始化。</p>
<p>当然，也可以是你根据需要创建不同的 Bundle 组合，给到他人来进行安装配置。</p>

![](https://cdnfile.sspai.com/2026/06/17/article/eee29ffeb4eee157c5af5716fda4e448.png?imageView2/2/w/1120/q/90/interlace/1/ignore-error/1)

<p>除了和 UniGetUI 强绑定的 软件 Bundle，UniGetUI 还支持导出一个 Powershell 脚本。</p>

![](https://cdnfile.sspai.com/2026/06/17/article/fc02c8ba3f9c2d2412baeb2a0eb72bc7.png?imageView2/2/w/1120/q/90/interlace/1/ignore-error/1)

<p>安装脚本还是完全和 UniGetUI 无关的，意味着你可以只交付这个 Powershell 脚本来交付你的软件包组合，可以减少你交付时的门槛（用户不用再装 UniGetUI 了，不是么）。</p>

![](https://cdnfile.sspai.com/2026/06/17/article/fea31c978f258ceb9172c31cbf416d40.png?imageView2/2/w/1120/q/90/interlace/1/ignore-error/1)

<h2>总结</h2>
<p>总体来说，我觉得 UniGetUI 是一个不错的包管理器，它提供的统一的更新 UI，可以帮助你确保软件都是最新的；同时，在聚合多个包管理器后，也可以让你的软件最大程度集中在一起管理。其自带的软件捆绑包则给这个软件提供了更多的可能性，除了个人使用，还可以用于团队管理、业务交付、软件开发环境构建等一系列需要批量配置的场景。</p>
