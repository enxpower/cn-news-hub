<p>Matrix 首页推荐</p>
<p>Matrix 是少数派的写作社区，我们主张分享真实的产品体验，有实用价值的经验与思考。我们会不定期挑选 Matrix 最优质的文章，展示来自用户的最真实的体验和观点。 </p>
<p>文章代表作者个人观点，少数派仅对标题和排版略作修改。</p>
<p>有时候难得两三人小聚，空暇时间却没什么东西打发时间，打把王者吧，时间不好控制，赶上更新一两百兆，流量和时间都没了。过年亲戚来玩，牌桌被大人们占了，微信小程序开房间还要看广告，麻烦得要死。桌游倒是开箱即用，但占地方不说，不熟悉的光看规则就头大了，难免各种村规/口胡。</p>
<p>现在各种在线游戏很多，但玩起来都缺少一种纯粹的乐趣，联机能玩的内容其实不多，毕竟大公司做游戏的目的就是为了赚钱，写代码、服务器这些都需要成本。我很喜欢玩的元气骑士、Minecraft，都是多人联机游戏里非常出色的游戏，但是上手总有这样那样的门槛，至少安装包下载是少不了的。</p>
<p>有没有什么地方能够完全不需要下载，没有任何门槛，点开就能联机和朋友一起玩的呢？这听起来像是天方夜谭，但 Parti 做到了。</p>
<h2>开端</h2>
<p>Steam 上有一个大名鼎鼎的游戏魔能系列，可以用 7 种不同的元素组合技能释放出不同的魔法，玩起来很有意思，可能很少有人知道它还有一个移动端的游戏《魔能方碑之巫》，或者叫《巫师也平板》，我小时候接触的就是这一版，玩起来非常有意思，剧情对话充满了冷幽默，也有很多隐藏技能需要自己发掘，威力强大，最有趣的是技能联动很有意思，各种元素组合出降雨术、炎爆术等等，玩起来很上头，可惜年代久远，十几年前的 32 位游戏现在已经没有多少设备能玩了。</p>
<p>最近我突然怀念起这个游戏，魔能的经典要素就是多人联机，混乱的法术导致的不同效果有时候能迸发巨大的威力，有时候也会把友军炸个半死，这才是多人联机的魅力，手游版本的魔能自然也不例外，但可惜网络环境导致基本上连不上官方服务器，也就根本没办法多人联机，我只能干看着那个灰色的联机按钮，放暑假找小伙伴们玩，也只能各玩各的存档，尝试了好多方法都不行。</p>
<p>要是能有自己做一个联机版本的方碑之巫就好了，我曾多次这样想过，但是一想到工作量，我就放弃了。</p>
<p>也是在很久很久以前，我曾经在 v 站发过一个自己写的基于 peerjs 的 web p2p 聊天工具 Peek，现在我自己也还在用这个工具传文件，很方便。后来我就在想能不能通过这个方式，做一个 p2p 的小游戏站点呢？不需要服务器，两个人扫码就能联机玩一些小游戏。Web RTC 其实完全可以做到，但是对于当时的我来说这样的项目精力消耗还是太大了，也就只是想想而已。</p>
<p>其实并非只是想想，由于当时的技术限制，我曾经做过一个 Node js 版本的联机平台，又是一个轮子造烂了的方向，还需要一台服务器才能运行，限制非常大，没搞多久我就放弃了。</p>
<p>直到我最近突发奇想，要是能让 peerjs 直接当作后端，解决复杂的联机协议问题，让开发者专注于编写游戏逻辑，那不是直接就能解放服务器，解放大脑了吗？</p>
<p>我脑海里顿时浮现出了一整个完整的协议大纲，一套自定义游戏代码机制，和一个不需要思考就能开发联机游戏的美好未来图景，当然，还有那稍微想想就能意识到大到令人头皮发麻的工作量。</p>
<h2>时代的进步</h2>
<p>但，这一切都随着 AI 大时代的到来被完美解决了，我只是稍微向 Codex 描述了一下我的初步方案，然后，这一切就这样水灵灵地出现了，Parti 诞生了，一个真正意义上的零门槛联机平台，一个支持完全自定义游戏代码的平台，乘着时代的风，来到了我的面前。</p>
<p>写代码？不存在的，整个 Parti 都是 Vibe Coding 出来的，Codex 把开发格式、联机协议、透明性等等处理的整整齐齐，当然，是在我的监督之下，否则我也无法想象这么大一套东西最终会变成什么究极屎山。不过目前来看，已经开发了十多个小游戏的情况下，Codex 的上下文窗口用量并没有显著的增长，这确实说明了 Parti 的架构策略小有成效。</p>
<p>得益于在开发之初，我就明确了 Parti 的最终形态，协议层、底层联机层、Web 层、自定义游戏层等被严格分离，确保了整体代码架构的稳定性，Parti 最终没有变成脱缰野马，而是一步一个脚印变成了我想象中的样子。</p>

<p>但需要注意的是，它要用系统自带浏览器打开才行，内置浏览器限制得太多了。不得不说，在 Web 如此牛逼的跨平台支持下，只要能打开网页的设备，基本上都能进行 Parti 游戏，感谢浏览器厂商多年的努力。</p>

<h2>想玩的游戏没有？可以自己做</h2>

![](https://cdnfile.sspai.com/2026/07/20/da686038176436b6363a5e06153883c5.png?imageView2/2/w/1120/q/90/interlace/1/ignore-error/1)

<p>Parti 也允许导入自定义房间。一个最简单的房间只有三个文件：</p>
<p><code>parti.room.json</code> 记录游戏名、人数、入口和权限<code>index.html</code> 管界面<code>room.worker.js</code> 保存规则和权威状态。</p>
<p>你可以从空白房间或模板开始，在编辑器里改，也可以直接导入 ZIP 或者 GitHub 仓库里的房间代码。现在 Parti 还支持了市场协议（基于 GitHub Issue），可以快速把自己开发的游戏发布到 Parti 中，让更多人能玩到。</p>
<p>当然，最简单也是我最推荐的方法，就是直接全部交给 AI，Parti 专门有一整套文档供开发者 / AI 阅读，只需要复制简单的提示词，就能让任何一个有些许 Agent 技能的 AI 直接帮你写出完整的游戏，稍微复杂一点，直接让 opencode 或者 Codex 代劳，连发布的活都省了。</p>
<p>比如用 Deepseek 网页版创建的德州扑克和井字棋游戏，啪地一下就搞定了。</p>
<h2>联机是如何实现的</h2>
<p>Parti 的联机原理也非常简单，它本质上就是通过 Web RTC 进行连接建立和数据传输，只是它把底层的脏活累活都处理好了，开发者只需要关心它的协议，api 如何调用，然后专注于自己的逻辑实现。</p>
<p>如果是服务器联机游戏开发，核心逻辑代码会运行在云端的服务器上，而在 Parti 中，由于完全没有服务器做后盾，这部分代码会通过 Web Worker 运行在房主设备上，并且通过房主连接所有人，承担大部分服务器的工作。</p>
<blockquote><p>房主：既然做好了开房间的准备，就要有承担更多责任的觉悟啊。</p></blockquote>
<p>这样做的好处是，Parti 完全不需要服务器支持，只要 Parti 网页还能访问，就能一直运行下去，一直联机，一直开房间。白嫖到底，这就是开发者的底气（误。</p>
<p>Parti 的房间分享功能也是内置的，开发者无需关心，就能自动获得，房间公开性设置、设置密码、二维码/链接分享、局域网公开等基础平台功能开箱即用。</p>
<h2>集大成者</h2>
<p>可以说，Parti 目前几乎把浏览器的能力发挥得淋漓尽致，它的联机能力来自于Web RTC，它的包加载能力依靠 Service Worker 实现，通过 Web Worker 运行游戏服务端逻辑，依靠 Canvas 实现画面渲染，动作传感器权限让重力感应成为可能，如果 Parti 早诞生几年，可能还没有如此广阔的设备支持如此多的新特性，甚至如果不是Codex，Parti如此之多的代码量，协议设计与适配器，开发工作量让人来实现也简直不敢想象。但现如今，随着手机厂商、浏览器厂商的努力，新特性不断普及，AI也越来越好用，正是这些天时地利一贯而出，让 Parti 最终横空出世。</p>
<h2>并非万能</h2>
<p>WebRTC 受运营商、NAT、公司网络和地区连接状况影响很大。有些网络环境就是连不上，或者延迟明显。遇到这种情况，可以试试切到局域网模式，或者让所有设备挂在同一个 Wi-Fi 下。</p>
<p>房主的浏览器要干权威计算的活，所以游戏期间房主不能把页面关掉。Parti 也没有竞技级的反作弊能力，它面向的就是朋友聚会、家庭娱乐和轻量社交。房主、玩家设备和房间代码都不能当成绝对可信的环境。说实话，要是跟朋友玩聚会游戏还要开挂，那大概也不是技术能解决的问题了。</p>
<p>一些内置游戏还在继续调。动作手感、网络表现、移动端布局、多人流程，都得靠更多真人试玩来磨。如果哪个游戏玩着有问题，欢迎直接反馈。要是你已经改好了，提个 PR 就更好了。</p>
<h2>开个房间再说</h2>
<p>现在 Parti 里有了《Endwell》，还陆续塞进了平台跳跃、炸弹竞技、坦克大战、你画我猜、狼人杀、斗地主和麻将。这些游戏打开浏览器就能玩，朋友离得近就递个二维码，离得远就发条链接。登录注册开会员？不存在的。</p>
<p>如果你心里也有一款一直想玩、却始终没能跟朋友联机上的游戏，可以翻翻代码和开发文档：Parti GitHub 仓库。也许 Parti 里的下一款游戏，就来自另一场迟到了很多年的联机。</p>
